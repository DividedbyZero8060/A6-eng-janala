Frequently Asked Questions

The difference between var, let, and const?
"var" keyword allows declared variables to be accessed in multiple scopes, including global scope. "let" keyword allows the variable to only be accessed in the declared scope but it can be changed indefinitely in the declared scope. "const" keyword makes sure the variable can store only one value during it's lifetime.
The difference between map(), forEach(), and filter()?
The map method transforms an array by applying a function to all of the elements and building a new array from the returned values. The forEach method iterates over each element of an array, just like a loop but does not do anything by default. The filter method filters out an array by applying a logic against each element of the array and building a new array from the returned elements
Explain arrow functions and how they are different from regular functions?
In arrow functions, we store the function in a variable. The parameters are stored in a parenthesis and after it comes an arrow consisting of an equal sign and a greater than symbol. After that follows a curly braced block in which all the computation goes. Arrow function expresses functionalities in a less verbose way
How JavaScript Promises work?
A promise in JS is an asynchronous action that may complete at some point and produce a value. It is able to notify anyone who is interested when the value is available. Promise can be returned from a fetch function or by the resolve method of Promise object. The result of promise can be accessed using it's then method. Then method registers a callback function to be called when the promise is resolved.
How closures work in JavaScript?
A function that references binding from local scope around it is called a closure. When called, function body sees the environment in which it was created, not the environment in which it was called. This mechanism allows variables defined in function to be accessed later.
